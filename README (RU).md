# EdTech Unit Economics Simulator

Интерактивный симулятор юнит-экономики онлайн-курса. Показывает, как completion rate, CAC, OpEx и repeat rate связаны между собой и куда утекают деньги когорты — от записи до повторной покупки.

## Зачем это нужно

В EdTech легко оптимизировать одну метрику в отрыв от других: снизить цену ради роста completion rate — и просадить gross margin; урезать OpEx на менторскую поддержку — и получить рост drop-off. Симулятор делает эти связи видимыми: двигаешь один параметр — сразу видно, что происходит с LTV:CAC и прибылью когорты.

## Что внутри

**Управляемые параметры:**
- Учеников в когорте
- Цена курса
- Completion rate (доля дошедших до конца)
- CAC — стоимость привлечения ученика
- OpEx — операционные расходы на ученика (менторы, поддержка, хостинг)
- Repeat rate — доля выпускников, покупающих следующий курс

**Расчётные метрики:**
- Drop-off (зеркало completion rate)
- LTV на ученика
- LTV : CAC — ключевой индикатор здоровья unit-экономики; ниже 3:1 подсвечивается как зона риска
- Gross margin
- Итоговая прибыль когорты

**Модель расчёта:**
```
Завершили курс   = Учеников × Completion rate
Купили повторно  = Завершили × Repeat rate
Выручка          = Учеников × Цена + Купили повторно × Цена
OpEx (итого)     = Учеников × OpEx на ученика
CAC (итого)      = Учеников × CAC на ученика
Прибыль когорты  = Выручка − OpEx − CAC
LTV              = Цена + Цена × Repeat rate × Completion rate
LTV : CAC        = LTV / CAC
```

Модель упрощённая (линейная, без учёта retention по месяцам и сезонности), задача — не точный P&L, а быстрая проверка гипотез и интуиция для продуктовых решений.

## Как запустить

Открыть `edtech_unit_economics_simulator.html` в любом браузере. Работает офлайн — все расчёты идут в браузере, ничего никуда не отправляется. Единственная внешняя зависимость — Chart.js, подтягивается с CDN при первом открытии (нужен интернет один раз).

## Стек

Ванильный HTML/CSS/JS + Chart.js для визуализации воронки. Без сборки, без зависимостей на бэкенд — можно встраивать куда угодно: в портфолио, в Notion-страницу, в презентацию.

## Возможные доработки

- Добавить retention по месяцам (cohort decay) для подписочных моделей
- Сравнение двух сценариев side-by-side (A/B по параметрам)
- Экспорт текущих значений в CSV/PDF для питч-дека
- Чувствительность прибыли к каждому параметру (tornado chart)
